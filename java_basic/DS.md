* **空间复杂度**：空间复杂度指的是一个算法在不同的输入下，根据输入大小而变化的空间大小。比如我一个算法只是开了一个固定大小的数组，输入内容不会影响这个数组的大小，那么这个算法的空间复杂度就是 `O(1)`；而归并排序中，常使用的辅助数组，它的大小就会随输入待排序数组大小的改变而改变，因此归并排序的空间复杂度是 `O(N)`

* **顺序表与ArrayList**

    1. 线性表的线性指的是逻辑线性，而不是存储空间的，因此链表、循环队列、顺序表都是线性表；二叉树不是线性表

    2. 一般情况下，链表的插入删除比顺序表要高，但是当链表没有保存尾结点引用时，就必须遍历到尾结点，但是顺序表就不用

* **顺序栈与链栈**

    链栈相比顺序栈，有点主要是插入是不需要扩容。（两个栈的实现方式，或者用C++的话说，容器适配器，都是提供的栈的操作，所以比较的不是栈和顺序表的特点）

* **树和二叉树**

    1. 一棵二叉树的前序遍历的得到的序列和后序遍历得到的序列恰好相反，则一定可以推出，这棵二叉树至多有一个叶子节点。前序遍历可以简单的概括为：根、左子树、右子树；后序遍历可以简单概括为：左子树、右子树、根，因此可以推断出，每个节点至多有一个孩子，这棵二叉树至多有一个叶子节点的情况，前序遍历和后序遍历的得到的结果相反

* **堆**

    * 堆的几个时间复杂度：

        * 向上调整建堆：`O(n * logn)`
        * 向下调整建堆：`O(n)`
        * 插入：`O(logn)`
        * 删除：`O(logn)`

* **排序算法**
    
    * 堆排序，通过的是堆顶元素只有一个，而堆的有序是以层为单位的，所以堆顶（只有一个元素的一层）一定是已经排好序了的，只需要将他放到末尾，就排好了一个元素，然后重新建堆往复。

        * 堆排序的时间复杂度是 `O(NlogN)`，不稳定，但是最坏的情况时间复杂度依然是 `O(NlogN)`
        
        * 所以，排升序建大根堆，排降序建小根堆
    
    * 选择排序，默认的方式是每次选出来一个最值（最大或最小），但是我们为了优化会一趟选出来最大值和最小值，虽然时间复杂度是相同的

    * 对于类似 `1, 3, 2, 4, 5, 6, 7, 8, 9` 的近似有序的序列，最快的排序是直接插入排序，时间复杂度近似 `O(N)`

    * 快速排序的非递归方式，可以使用栈模拟边界信息在栈帧中的传递，也可以使用队列模拟二叉树的层序遍历
