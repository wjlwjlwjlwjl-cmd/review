5. 哈希函数
（1）除法散列法：也叫做除留余数法，对于哈希表的大小有限制，尽量不是某些数的幂，否则会导致相应进制表示的指数位数的个数相同的数
都会产生哈希碰撞
（2）乘法散列法：通过一个小数A与key相乘，去小数部分，再与M相乘（哈希表大小），得到下标
（3）全域散列法：取一个较大的指数P，a属于[1, P], b属于[0, P]，(a * key + b) % P % M，就是函数表达式，但是这构成了
一个散列函数组，每次初始化时选择一个，后续哈希都通过它进行
6. 处理哈希冲突
（1）开放定址法：主要有三种方法，线性探测、二次探测、双重探测
## 线性探测
从发生哈希冲突的位置向后探测，直到找到能存储key的位置。但是比如hash0、hash1、hash2都已经存储，hash3还未存储，
那么后续映射到hash0~3的key都会争抢hash3的位置，这种现象叫做群集/堆积
## 二次探测
在发生哈希冲突的位置双向查看二的幂的距离是否可以存储（1 ~ M/2）
## 双重探测
为了解决线性探测的堆积问题，使用第一个哈希函数计算结果冲突时，使用第二个哈希函数计算出检查的步长通过这个步长去寻找能够存储的位置
（2）链地址法
链地址法，哈希表中存储指针，当有数据存储时，都在该位置存储的链表中存储；如果出现某个链表过长的情况，可以采取全域散列法，如果依然有这种情况，那么可以到达一定
阈值就让链表转化为红黑树
