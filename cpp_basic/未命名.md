###  第二章：编译和链接

#### 1.1 被隐藏了的过程

<strong>构建</strong>，编译和链接合并到一起的过程

```cpp
#include <stdio.h>
int main(){
    printf("hello world\n");
    return 0;
}
```

在Linux下，我们使用GCC来编译，只需要`gcc hello.c && ./a.out`，但是这个过程可以分为以下步骤：

* 预处理
* 编译
* 汇编
* 链接

##### 预编译

对于C程序，预编译文件的后缀是`.i`；对于C++程序，预编译的后缀是`.ii`，第一步预编译可以理解为使用gcc的以下命令

```shell
gcc -E hello.c -o hello.i
```

预编译阶段主要完成以下工作

1. 删除所有宏，并在使用处完成展开
2. 递归展开所有包含的头文件
3. 删除所有的注释
4. 添加行号和文件名表示，以便编译报错时显示文件名和行号
5. 处理所有条件预编译指令，如`if` `ifedf` `endif` `elif`等
6. 保留所有`#pragma`编译器指令

##### 编译

通过进行词法分析、语法分析、语义分析及优化（现代C++各种模板、模板元编程等等都是在这个期间完成运行），生成汇编代码文件，这个过程相当于以下命令

```shell
gcc -S hello.i -o hello.s
```

###### 词法分析

1. 源代码程序被输入到扫描机，简单的进行一系列词法分析，将字符串序列分割成一系列的记号
2. 这些记号通常包含：关键字、标识符、字面量（数字、字符串...）、特殊符号（如加号、等号）
3. 扫描器会将标识符存放到符号表，数字、字符串常量存放到文字表（C++的inline修饰变量，C语言的extern，和声明定义分离的方法都是为了解决这里标识符在符号表中冲突的问题）

###### 语法分析

语法分析器会根据词法分析器生成的记号进行语法分析，将源文件中的表达式生成一棵以表达式为节点的语法树，在此时，如果出现表达式不合法，比如括号匹配问题、表达式中操作符缺少，都会在这个阶段产生错误 

 

> 语法分析和词法分析都有现成的工具，只需要更改规则，就可以完成对不同高级语言的处理

###### 语义分析

这个阶段完成的是对表达式语法层面的分析，而且分析的是静态语义（即编译时能确定的，比如类型的转换，动态语义的问题比如除零错误，这就是运行时才能知道的）  

比如浮点数赋值给整形，其中的隐式类型转换，就是在这个过程完成的；比如把浮点数赋值给指针，这个过程的不匹配错误，也是这个过程报出的  

经过语义分析后，会为语法树中的每一个表达式表明类型，需要隐式类型转换的也会在树中插入相应的转换节点

###### 中间语言生成

1. 在生成目标代码之前，**源码级优化器**会进行优化，比如一些可以在编译器就确定的值(`2 + 8`这种字面量的运算)，就会在此时被优化
2. 但是并不是直接在语法树上进行，而是会生成中间代码，他是语法树的顺序表示，虽然很接近目标代码，但是两者之间还有差别，比如不是和平台相关的，没有数据尺寸、变量地址、寄存器名称等
3. 中间代码有很多类型，比如三地址码，P-代码等
4. 这种工作方式使得编译器可以被分为前端和后端两个部分，前端负责生成机器无关的中间代码，后端负责生成目标机器代码。这样在跨平台的编译器上就可以实现一个前端对应多个后端

###### 目标代码的生成与优化

1. 这一步部分的工作有编译器后端来完成，其又有两个组成部分：目标代码生成器和目标代码优化器
2. 目标代码生成器负责将中间代码，根据机器的寄存器、字长、整形情况、浮点数数据类型等，生成目标机器代码；目标代码优化器负责对上一步骤的结果优化，比如寻址方式的优化、使用位移代替乘法、删除多余指令等等

##### 汇编

这一步要比编译简单，因为几乎每一条汇编语言都对应一条机器指令（这也是我们为什么常说对应多条汇编指令的编程语言不是线程安全的），汇编只需要完成一一对应即可，这也是“汇编”这个名字的由来。  

经过预处理、编译、汇编之后，生成了目标文件

```shell
gcc -c hello.s -o hello.o
```

无论我们想要访问变量，还是函数，都需要拿到地址，但是当程序修改的过程中，这些都要重新计算，这就是**重定位**

汇编器在每次汇编时就会重新计算使用到的变量、函数的地址，并修改所有使用了这些变量、函数的指令中的地址。

但是当程序分为多个模块时，就需要解决多个模块怎样合成一个程序的问题。要解决这个问题，其实就是要解决模块间的通信问题，比如对于C/C++这些静态语言，主要就有函数调用、变量访问这两种方式完成模块间的通信，而这就需要解决模块间符号的引用的问题。就像拼拼图一样，一个文件中定义的某个符号就是一块拼图凸出来的一块，在另一个目标文件中对其又进行了引用，这就是这块拼图凹进去的部分，两者拼在一起，才能够形成完整的程序，这里就引入了我们链接的概念
