# 线程简介
## 什么是线程
1. 线程是进程中的一个执行路线，或者说，线程是一个进程内部的控制路线
2. 一个进程至少有一个执行线程
3. 线程在进程内部运行，本质是在进程的地址空间中运行；透过进程虚拟地址空间，可以看到
进程的大部分资源，将这些资源合理分配给每个执行流，就形成了线程执行流
## 线程的优点
1. 相比创建一个进程，创建一个线程更轻量，比如不需要新的地址空间，不需要建立额外的页表等
2. 线程的切换更加轻便。进程的切换需要带走进程的上下文，同时TLB的缓存还有三级高速缓存
中的内容全部作废，所以在刚完成切换的一段时间中，cpu的效率是非常低下的
3. 能充分利用多处理器的可并行数量
4. 对于I/O密集型应用，能够将I/O操作重叠，让线程等待不同的I/O操作；对于计算密集型应用，可以将计算过程
分到多个处理器上实现
## 线程的缺点
### 1. 性能损失：
对于较少遇到阻塞的计算密集型线程，往往难以与其他线程共享同一个处理器；对于计算密集型线程数量多余可用的处理器
数量的情况，会有较大的性能损失，因为有大量的同步和调度开销
### 2. 健壮性降低
线程安全是需要保证的，不合理的分配资源可能对整体性能有很大影响
### 3. 缺乏访问控制
进程是访问控制的基本粒度，一个线程出问题会导致整个进程受影响。
# 进程和线程
## 两个基本单位
* 进程是资源分配的基本单位
* 线程是调度的基本单位
## 线程“私有”的资源
1. 线程id
2. 一组寄存器，线程的上下文数据
3. 栈
4. errno
5. 信号屏蔽字
6. 调度优先级 
## “线程的公有资源”
1. 文件描述符表
2. 每种信号的处理方式(SIG_DFL, SIG_IGN, 自定义的信号处理函数);
3. 当前工作目录
4. 用户id和组id
# Linux线程控制
## 线程产生
```cpp
int pthread_create(pthread_t* thread, const pthread_attr_t* attr, void*(*start_routine)(void*), void* arg);
```
thread，带出线程id，attr设置线程属性，没有设置则使用默认属性；start_routine设置线程启动后执行的函数；arg，传递给线程启动函数的参数

```cpp
pthread_t pthread_self();
```
获取线程自身的线程id
## 线程查看
```shell
wang@DESKTOP-RSKTHP7:~$ ps -aL | head -1 && ps -aL | grep a.out
    PID     LWP TTY          TIME CMD
    1353    1353 pts/2    00:00:00 a.out
    1353    1354 pts/2    00:00:00 a.out
```
* 线程id是pthread维护的用来在进程内表示线程唯一性的id，是进程级的，内核并不认识
* LWP是真正的线程id，使用pthread_self得到的其实是一个虚拟地址，其中存放了线程的基本信息，包括线程id、线程栈、寄存器等属性
* 上面检索出的两个线程中，有一个PID和PWD相同，这就是主线程，他的栈位于地址空间中；而子线程的栈位于共享区，因为pthread作为动态库被加载到了共享区，而子线程的栈又在这里被维护
## 线程终止
* 函数调用return返回，不过对于主线程不适用，相当于exit
* pthread_exit，终止所在线程
```cpp
void pthread_exit(void* value_ptr);
```
通过value_ptr带出推出信息，注意不要使用局部变量
* pthread_cancel，终止其他线程
```cpp
int pthread_cancel(pthread_t thread);
```
## 线程等待
* 为什么要线程等待？
1. 线程退出之后，在共享区中维护的堆栈并没有释放
2. 新的线程并不会复用尚未释放的老的堆栈空间
* 使用pthread_join等待
```cpp
int pthread_join(pthread_t pid, void** retval);
```
根据线程退出的方式，retval会议不同的方式带出线程退出信息

1. 如果线程通过return退出，则retval会带出线程返回值
2. 如果线程使用pthread_exit退出，则retval会带出传给pthreadexit的参数
3. 如果线程被pthead_cancel退出，则retval会带出PTHREAD_EXIT
4. 如果retval设置为null，则不会带出返回值
## 线程分离
如果我们不想再对一个线程关心，那么我们就可以和这个线程断绝关系，也就是分离，使用pthread_detach函数
### pthead_detach
```cpp
int pthread_detach(pthread_t pid);
```
pthread_detach既可以有主线程调用，也可以有子线程本身调用，但是一个线程如果已经分离出去了，就不能再等待了
