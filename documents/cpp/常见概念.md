## 基础概念
### 缺省参数
缺省参数需要注意的点有两个：
1. 缺省参数的声明和给定<br>
在函数声明中，缺省参数必须从右向左给定；使用函数时，参数必须从左向右依次传入，中间不允许出现跳跃情况
2. 缺省参数的出现位置<br>
在声明和定义分离的情况下，缺省函数只能出现在声明中，不能出现在定义中；但是不分离的情况下，因为函数的定义这时等价于声明，所以可以直接出现
### 函数重载
c++允许在同一作用域中出现同名函数，在形参不同时（可以时形参数目不同，也可以时形参类型不同），形成函数重载，表现出多态的行为。
<strong>需要注意的是，返回值不同不能作为判断依据，也很好理解，返回值时函数调用完的产物，但是不调用又不知道返回值的区别，编译器就犯难了</strong>

### 引用 
1. 引用其实就是给变量取了别名，对于引用对象，不会再为它开辟内存空间，其与被引用对象共享内存空间（可通过取地址查看），但是引用在使用时需要注意这些问题
* 引用在定义时必须初始化
* 一个对象可以有多个引用
* 与java不同，引用一旦初始化后，不能再改变指向
```cpp
int a = 10;
//int &b;
int &b = a;
int &c = a;
```
#### const 引用
1. 使用范围：对于常量，或者是像临时变量这种具有常性的变量，需要使用>const
2. 权限问题：权限可以缩小，但是不能放大，对于const引用，可以引用普通对象，但是普通引用不能引用常量
3. 指针和引用的关系:<br>
* 引用不开辟内存空间，但是指针需要开辟存放地址的空间
* 引用可以直接访问被引用对象，但是指针需要先解引用
* sizeof结果上，引用返回的是被引用对象的sizeof结果，而指针是该平台指针大小 
* 初始化问题：引用在定义时必须初始化，指针只是建议
* 改变指向问题：引用在确定引用对象后不能改变，但是指针在指向一个变量后可以随时改变指向对象
```cpp
int x = 10;
const int& y = x * 3;
const int& z = 10;
const int& m = 10;
const int& n = x;
```
### inline 关键字
对于inline修饰的函数，编译器会选择性的将其在被调用的地方展开，不再为该函数常见函数调用栈帧，类似与C语言的宏展开，不过需要注意的是，对于递归和代码量大的函数，编译器会给予忽略<br>
另外，内联函数不建议声明与定义分离，因为内联函数只是展开，不会开辟地址，所以在链接时会报错
### nullptr
* nullptr: c++中的nullptr是一种特殊类型的字面量，可以转化为其他任意的指针类型，但是不能够转化为整数
* NULL: C语言中的NULL是一个宏，可能是数字0，也可能是(void*)0，可以在stddef.h中查看，具体是哪个，取决于是否定义__cplusplus
* 综上，NULL对于c++这种提供了函数重载的语言来说是不好的（因为不能对NULL强制类型转换），例如以下代码
```cpp
void func(int* a)
{
    std::cout << "int*" << std::endl;
}

void func(int a)
{
    std::cout << "int" << std::endl;
}
```
使用NULL就可能会导致调用问题，甚至会可能直接编译报错，但是nullptr不能饮食类型转换为整形，所以就避免了这样的问题
## 类和对象
### 类域的概念
类定义的自己的作用域，外界访问需要通过‘：：’操作符，否则编译器默认不会到类域查找，只会在全局作用域；对于类中声明的函数，分离实现时，也需要指明类域。除此之外，类域也具有特殊的性质，就是编译器编译顺序，类域外，A函数在B函数前，但是A函数又调用了B函数，就会编译报错；类域中，查找顺序不再是简单的从上到下，而是整个类域都可以
### 类的实例化
* 所谓实例化，就是根据类定义中成员函数和成员变量的声明，分配相应的内存空间进行存储（成员函数有些特殊，这里不对成员函数的函数指针进行存储，后面还会提到，函数被编译后，形成一段指令，存储在代码段）
* 对于大小，其实和计算C语言的结构体没有特别大的区别，需要注意，对于空的类（没有成员函数和成员变量）以及只有成员函数的类，也会用一个字节来表示这个类的存在，不会不开辟内存空间
```cpp
class A
{
public:
    void f1()
    {
        std::cout << "f1" << std::endl; 
    }
    void f2()
    {
        std::cout << "f2" << std::endl;
        std::cout << _a << std::endl;
    }
private:
    int _a;
}

int main()
{
    A* a = nullptr;
    a->f1();
    a->f2();
    return 0;
}
```
* 以上程序，就会报错，原因是编译后，函数形成的指令可存储在代码段，但是并未实例化A，以为这没有对_a分配内存空间，这样在调用f2时就会段错误
### this指针
* 为了表明成员函数表明调用的是哪个实例化对象中的成员变量，c++引入了this指针，成员函数内部对成员变量的操作其实都是被这个隐式的this指针完成的，但是this指针不允许在形参和实参中出现，只可以在函数体中显示调用
* 对于非静态成员函数，每个成员函数都有自己的this指针，这个指针可以为空，不过不能直接赋空，可以强制类型转换
```cpp
class A
{
public:
    void f(int a)
    {
        this->a = a;
    }
private:
    int a;
}
```

### 构造函数
* 类名+参数列表，不写返回值，这就是构造函数
* 默认构造函数，包括三种
  1. 全缺省构造函数
  2. 无参构造函数
  3. 编译器自己生成的构造函数

总结起来其实就是不需要传参就能够使用的构造函数，但是默认构造函数只能存在一个，因为1、2如果自己实现了编译器就不会实现3，同时1、2也不能同时存在，因为存在调用二义性
* 对于内置类型，编译器生成的构造函数可以直接处理，但是对于自定义类型，则会去调用其默认构造函数，如果没有则报错
#### 初始化列表
* 无论是否显示实现初始化列表，都会先走初始化列表，然后才是构造函数函数体中的操作
* 每个成员变量在初始化列表中只能出现一次，并且对于引用、const成员变量、和没有默认构造的类类型变量，都必须在初始化列表初始化
* 初始化列表初始化顺序：只取决于声明的顺序，与初始化列表中的顺序无关
* 构造函数初始化顺序，有初始化列表优先按照初始化列表初始化，没有初始化列表，如果有缺省值，就用缺省值初始化，对于没有缺省值的，对于内置类型，可能初始化为默认值0，也可能初始化为随机值，对于自定义类型，如果没有默认构造，直接报错，对于引用类型和const成员变量，直接报错
### 析构函数
* ~+类名+() 就是析构函数，需要注意的是，对于内置类型，编译器自己生成的析构函数不做处理，因为这些栈上的内存会随函数栈帧的销毁而销毁；而自定义类型会调用他的析构函数
* 与构造函数相比，一个类只能有一个析构函数，如果没有申请资源，也可以不写析构函数
* 对于类中类，定义顺序在前的后析构
### 拷贝构造函数
* 是构造函数的重载，调用时对象还未实例化，参数要求：第一个参数必须是这个类的引用，后面还可以加参数，但是都必须有缺省值，否则虽然不会报错，但是编译器仍然会调用自己生成的拷贝构造函数
* 编译器默认生成的构造函数是浅拷贝，即只是简单的完成值的拷贝，对于指向资源的指针，我们还需要深拷贝，即开辟另一块相同的资源，再让新的指针指向这块资源，这就是我们需要自己实现拷贝构造的情况。
* 简单判断是否需要自己实现拷贝构造：查看是否有析构函数且还在内部显示释放资源，若没有，则使用默认即可
* 如果返回自定义类型的对象时，使用的是传值返回，那么会调用拷贝构造，使用传引用返回时，如果返回对象的声明周期和返回的函数相同，那么会造成“野引用”的问题
### 赋值运算符重载
#### 运算符重载
* operator+重载运算符，就构成了运算符重载；如果这不是成员函数，那么参数个数等于操作数个数，比如一元操作符需要一个参数，二元操作符需要两个参数；但是对于成员函数类型，参数的个数就比操作数个数少一个，因为第一个参数会默认传给隐式的this指针
* 前置++和后置++，两者的函数名是相同的，但是为了有所分别，规定后置++在参数列表（只是为了分别）加上一个int形参
* 规定，不能够重载原来不是操作符的符号
* 规定，不能够重载这些操作符'.*', 'sizeof', '::', '?:', '.'
* .*操作符表示通过对象调用成员函数指针
```cpp
class A
{
public:
    void func()
    {
        std::cout << "func" << std::endl;
    }
};
typedef void(A::*PF)(); //定义一个成员函数指针
int main()
{
    PF* pf = &A::func; //cpp规定，加上&才能够取到成员函数指针
    A a;
    (a.*pf)(); 
    return 0;
}
```
#### 赋值运算符重载
* 赋值运算符重载建议返回当前类型引用以完成连续赋值，同时传参时建议接受const引用，以满足临时对象和减少拷贝的需求
* 对于编译器自己生成的赋值运算符重载，进行的是浅拷贝，如果没有申请的资源，那么编译器生成的已经足够使用，对于需要自己实现的情况，判断方法和拷贝构造相同
* 赋值运算符重载如果不在类中显示实现，编译器就会自己生成，这意味这赋值运算符重载只能在类域中作为成员函数实现，否则自己的就会和编译器的发生冲突
* 实现细节，可以通过this指针进行判断，避免自赋值的情况
### 取地址运算符重载
#### const成员函数
1. 使用方法，在参数列表后加上const
2. 效果，const作用于this指针，让this指针从（假设日期类）Date* const 到const Date* this，表明该成员函数不得修改任何成员变量
#### 取地址运算符重载
1. 其实一般情况下编译器生成的已经足够使用，但是如果有一些需要，比如不希望外界拿到自己某个类的地址，就可以胡乱返回一个指针
2. 分为普通取地址重载和const取地址重载

### 类型转换
* 允许内置类型转换为自定义类型，需要在构造函数中进行实现，也可以将一种自定义类型转换为其他自定义类型
* 这种转换是隐式的，只要提供了相应的构造函数，但是如果不希望存在这种转换，只需在相应构造函数前加上explicit关键字即可
### 静态成员
* static修饰的成员变量和成员函数，都是一个类的静态成员
* 对于静态成员变量，不能在类中初始化，也不能通过在声明时给予初始值的方式进行初始化，只能在类外进行初始化
* 静态成员变量的存储：不存储在类中，存储在静态区中
* 静态成员函数可以访问静态成员变量，但是因为静态成员函数没有this指针，所以不能访问非静态成员，外界想要访问静态成员变量，通过::或者.就可以进行访问。
* 静态成员不属于某一个具体的实例化的类，一个类的所有非静态成员函数都可以访问这个类的所有静态成员变量和函数
* 静态成员也受到public、private、protected这些访问限定符的限制
### 友元
* 用来突破类域的限制，分为友元类和友元函数两种，可以放在类的任何位置
* 友元不具有传递性，是单向的，也可以是一对多的关系
### 内部类
* 将一个类的定义放在另一个类当中，这样被“包”起来的类就叫做内部类，它被默认为是外部类的友元类，能够访问外部类的所有成员函数和变量
* 定义内部类时，需要通过'::'突破外部类的类域来进行访问
### 匿名对象
* 声明周期只在当前行，主要使用于临时使用的场景，不需要对他取名字
比如下面的代码就是匿名对象的使用场景之一
```cpp
class A
{
public:
    A(int a = 0)
        : _a(a)
    {}
    int Solution()
    {}
private:
    int _a;
};
int main()
{
    //A a(); 这样写，编译器无法识别这是函数声明还是对象定义
    A (1).Soluiton();
    A().Solution();
    return 0;
}
```
## 内存管理
### 操作符new和delete
* c++内存管理主要通过new和delete两个操作符完成。
* new的工作过程是，先调用operator new这个全局函数来申请空间，之后在这块空间上调用构造函数，返回指向这块空间的指针；delete的工作过程是先在获得的空间上调用析构函数，随后通过operator delete来释放空间。当处理失败时，两个函数都会抛出异常，至于new[]和delete[]就是批量处理.
* 与malloc和free的区别
1. malloc和free是函数，new和delete是操作符
2. malloc只是申请空间，没有执行构造；free只是释放空间，没有调用析构
3. malloc的返回值使用需要从void*强转，而new可以直接使用
4. malloc失败，返回NULL，但是new抛异常
5. malloc要手动计算开辟空间的大小，而new可以根据后面的类型自动完成开辟大小的确定
* 对于堆的大小，只有操作系统决定，只能动态分配；栈的大小一般较小，也可以通过_alloca动态分配，不过不是通过free和delete释放
* 频繁的new和delete会导致堆的碎片化，但是栈不存在这个问题，因为栈是自动管理的
* new其实内部调用了malloc，delete其实内部调用了free，所以两者也需要头文件malloc.h支持，只不过一般这个头文件已经被其他各种文件包含了
### 定位new
* 其实定位new就是把一块已经申请的内存通过构造函数实例化出对象，在内存池中有应用
* 使用方法 new(ptr_to_mem)class_name(initializer_list)
示例如下
```cpp
class A
{
public:
    A(int a) : _a(a) {}
    ~A(){}
private:
    int _a;
};
int main()
{
    A* pa = (A*)malloc(sizeof(A));
    new(pa)A(10);
    pa->~A();
    operator delete(pa);
    return 0;
}
```
可以认为定位new就是把new和delete的工作交给自己手动分步完成
## 容器适配器
priority_queue，优先级队列就是一种容器适配器，默认实现为大根堆，对于自定义类型建立大根堆，直接push即可；如果想要建立小根堆，则需要传入仿函数，并指明底层实现容器，默认是vector。<br>
容器适配器就是将一个类的接口，转换成用户希望的另一个类的接口
比如常使用的stl中的list和queue，就是两个容器适配器，默认底层容器是deque，类似于一个二维数组，具有头部、尾部操作效率高，同时扩容不用移动大量元素，但是遍历效率低，因为需要经常判断是否到达某个小区间的边界（迭代器设计也相对复杂，既需要有指针指向中控节点，有需要有指向小区间起始位置和当前位置的指针）
这样的特点就完美符合链表和队列的需求
1. 栈、队列的元素操作都在头尾，相比vector具有较少挪动元素的优势
2. 两者都不需要进行遍历操作，避开了deque遍历效率低的特点
3. 每个节点指向的空间是一段一段的连续空间，不是list高度碎片化的存储，内存利用率更高
## 模板
### 非类型模板参数
1. 模板参数中，跟在class或者typename后的表明类型的参数，是模板参数，其他类型，作为常量，可以在模板中使用，就是非类型模板参数
2. 浮点型、类对象、字符串都不允许作为非类型模板参数，非类型模板参数必须在编译期就能够确定结果
3. 模板参数中也可以只有非类型模板参数没有类型模板参数
## 特化
### 函数模板的特化
对于一些函数模板，比如基础的比大小，对于内置类型和已经重载过相应符号的自定义类型，都是基本满足需求的。但是一些特殊情况，比如需要比较的两个对象都通过指针的方式进行提供，这时就需要特化了（否则按之前的逻辑会出现比较地址高低的情况）
<strong>基本格式：</strong>，template+<>，返回值与函数名不变，在函数名后加上需要特化的类型，参数列表中是需要比较的形参，不过需要注意的是，要和模板参数的基础参数类型完全相同，否则编译器可能会出现奇怪的错误

```cpp
template <class T>
bool less(T val1, T val2)
{
    return val1 < val2;
}

template<>
bool less<Grade*>(Grade* g1, Grade* g2)
{
    return *g1 < *g2;
}
```
### 类模板的特化
对于类，可以分为两种情况，第一种是全特化，对所有模板参数都进行明确限定，不再有其他可能；另一种是偏特化，可以有两种表现形式，一种是对部分模板参数明确限定，其余可以改变，另一种是进行进一步限定，例如传值和传引用传指针
例如这种情况，记录并保存两个数据
```cpp
//未特化的类模板
template <class T>
class Data
{
public:
    T _data1;
    T _data2;
};

//全特化
template<>
class Data<int, char>
{
    //...
}

//偏特化1
template<class T1, class T2>
class Data<T1&, T2&>
{
    //...
}
//偏特化2
template<class T>
class Data<int, T>
{
    //...
}
```
### 模板的分离编译
* 分离编译，就是各个源文件各自经过预处理、编译、汇编之后，统一生成目标文件，再经过链接之后生成一份可执行文件，这个过程头文件不参与，因为在预处理阶段就已经完成了展开
* 模板如果在一个头文件中声明，在一个源文件中进行定义，再在另一个源文件调用，就会出现链接报错
```cpp
//add.h
template <class T>
T add(T val1, T val2);

//add.c 这里在编译之后，因为没有调用，所以不会实例化出代码
template <class T>
T add(T val1, T val2)
{
    return val1 + val2;
}

//main.c 在连接过程中，编译器才会寻找add的地址，但是add.o又没有实例化出代码，所以就会发生链接报错
int main()
{
    add(1, 2);
    return 0;
}
```
* 解决方法，可以将声明和定义放在一个.h或者.hpp文件中，也可以在模板定义的位置显示实例化（理论可行）
## 继承
### 三种继承方式
常用public继承，此外还有protected和private继承，三种方式的成员处理如下
|继承方式 | public | protected | private |
|--------|--------|----------|--------|
|public成员|public|protected|private|
|protected成员 |protected|protected|private|
|private成员|不可见|不可见|不可见|
### 继承类模板的按需实例化
```cpp
template<class T>
class stack: public vector<T>
{
public:  
    void push(T val)
    {
        vector<T>::push_back(val);
        //push_back(val); 找不到该函数
    }
    void pop()
    {
        vector<T>::pop_back();
    }
    T top()
    {
        return vector<T>::back();
    }
private:
    vector<T> con;
};
```
上面的代码通过继承stl中的vector模板来封装实现栈，在使用时会实例化stack和vector，但是模板是按需实例化，其中的push_back、pop_back、back这些成员没有实例化，所以要指定类域，实例化出相应成员访问
### 派生类可以赋值给基类的指针或者引用
发生类似与切片的行为，指针或者引用的都是派生类中继承基类的一部分
### 继承作用域的隐藏关系
对于基类和派生类中的同名成员，派生类会阻止对基类同名成员的直接访问，如果需要访问的话，需要通过"基类::基类成员"显示访问；对于成员函数，只要名字相同，就构成隐藏，例如以下代码
```cpp
class A{public: void func(){}};
class B: public A{public: void func(int a){}};
int main()
{
    B b; b.func(10); 
    b.A::func(); //b.func();
    return 0;
}
```
### 派生类的四个默认成员函数
* 构造函数，派生类实现必须显示调用基类的构造函数，除非基类有自己的默认构造函数
* 拷贝构造函数，必须显式调用基类的拷贝构造函数（这里就呼应了前面说的派生类可以赋值给基类的指针或者引用，发生“切片”）
* 赋值运算符重载，派生类和基类的赋值运算符重载构成隐藏关系，所以要指定基类类域并显示调用基类的赋值运算符重载
* 析构函数，只需要正常实现，会在完成子类析构之后自动调用基类析构函数，完成基类部分的清理工作
### 实现不可继承的类
* C++98后，通过将构造函数设置为private，派生类就无法实例化的方式
* C++11后，引入final关键字，被final修饰的类不可被继承
```cpp
class Base final
{};
```
### 友元关系和静态成员
两者在继承体系中，友元不能被继承，只在当前类中有效，静态成员在继承体系中有且只有一个
### 两种继承模型
* 单继承，一个派生类只有一个直接继承的基类
* 多继承，一个派生类有两个及以上的直接继承的基类，在内存上，先继承的类的成员在前面，后继承的类的成员在后面，最后是派生类自己的成员。这样就引入了菱形继承的问题，类B、C继承自类A，而类D又继承了B和C，这样在内存中，B的成员在前，C的成员在后，最后是D自己的成员，这样就导致D有两份相同的A的成员，具有数据冗余和二义性的问题。虽然可以通过指定类域的方式避免二义性，但是数据冗余的问题无法解决
* virtual继承可以解决数据冗余和二义性问题，但是依然不建议使用菱型继承，比如上面的情景，就可以让B、C virtual继承类A
### 继承和组合
* public继承一种is-a的关系，每个派生出的派生类都是另一种基类；组合是一种has—a关系，一个类中有另一个类。
* public继承，因为基类内部的实现细节对派生类可见，并且对基类的修改往往会影响派生类，所以破坏了基类的封装性，增加了代码耦合性；而对于组合而言，需要有良好的接口暴露使用，内部细节并不可见，所以代码耦合性低
* 根据实际情况，优先使用组合，在适合使用继承的is-a的情况下，再使用继承
## 多态
* 多态进行的两个条件：
1. 通过基类的指针或者引用调用虚函数
2. 虚函数完成了重写/覆盖
* 虚函数：
1. 有virtual修饰的成员函数，并且在派生类中有完全一样的函数（返回值、名称、参数类型和数目完全相同）的函数，就是虚函数
2. 多态中，派生类的虚函数继承了基类的声明，重写的是实现，具体可以看下面的例子
```cpp
class A{public: vitual void func(int val = 1){cout << val << endl;}
void test(){func();}};
class B: public A{public: virtual void func(int val = 2){cout << val << endl;}}
int main()
{
    B b; b.test();
    return 0;
}
```
3. 虚函数可以在类中定义，在类外实现，但是在类中时需要加上virtual关键字，并且在类外定义时不能加
4. 并不是所有成员函数都可以实现为虚函数，比如静态成员函数，在继承体系中只有一份，并不属于具体的某一个类，这时，静态成员函数就没有this指针，下面提到的虚函数表，没有this指针也就无法拿到，因此也就无法完成重写，无法实现多态
5. 基类的引用去引用了派生类对象后，通过基类对象调用虚函数，调用的依然是基类的虚函数，因为通过的是对象而不是指针或者引用，做这种类型的判断选择题时要注意抠字眼
### 协变
基类的虚函数返回基类的指针或者引用，派生类的相应虚函数返回派生类的指针或者引用
### 将析构函数实现为虚函数
* 编译器对虚构函数的处理：析构函数函数名会被编译器处理为destructor，因此尽管编写代码时派生类和基类的析构函数名称不一致，但是编译之后仍然是一致的，所以只要基类虚函数有virtual修饰，就构成了重写
* 为什么建议基类的析构函数实现为虚函数：现在有一个基类A，派生出了B，B中申请了一笔资源，现在处于多态的需求通过A的指针或者引用调用B的虚函数，最后通过A的指针或者引用释放，这时，如果A的析构函数不是虚函数的话，就只会调用A的析构函数，导致B中资源未被释放而泄露；但是如果如果是虚函数，那么就会调用B的析构函数，前面我们又说过，继承体系中，派生类的析构函数会在进行完派生类的析构之后自动调用基类的析构函数，所以这样就完成了派生类和基类各自的资源释放
### 重载、重写/覆盖、隐藏
1. 重载: 重载是两个函数在同意作用域中，具有相同的函数名，不同的函数参数（类型、个数），从而体现出不同行为，返回值是否相同均可
2. 重写/覆盖：分别在继承体系的两个作用域中，具有相同的函数名、参数、和返回类型（协变除外），都是虚函数，并且子类中的虚函数重写了父类中虚函数的实现，从而在使用基类的指针或者引用调用重写/覆盖的虚函数时体现出不同的行为
3. 隐藏：在继承体系中，同名的成员函数不构成多态就构成隐藏，对于同名的成员变量，也构成隐藏
### 多态的实现原理
1. 先来看下面的代码
```cpp
class A
{
public:
    virtual void func()
    {}
private:
    int _data; char _ch;
};
```
32位平台下，这个类的大小是12个字节。按照内存对齐的规则，这个类应该是8个字节啊！其实这就是多了一个虚函数表指针的原因。对于存在虚函数的类，都会有至少一个虚函数表，其中这个表中存放着指向这个类要调用的虚函数的函数指针。对于多态来说，通过基类的指针、引用调用的虚函数并不在编译时确定地址，而是在运行时，去相应的类中的虚函数表中获取相应的虚函数地址，进行调用，所以能够实现多态的效果
2. 对原理的理解，两道笔试题
```cpp
class A{public: virtual void func(){std::cout << "A::func";}};
class B: public A{private: virtual void func(){std::cout << "B::func";}};
A* pa = (B*)new B; pa->func();
```
这里随后调用的时A的func，虽然B中，func被放在了私有函数，但是满足了重写的条件，这样也没有去调用禁止访问的B的私有成员函数，而是去B的虚函数表中获得了A中func的调用地址，完成调用
```cpp
class A{
public: A(){test();}
    virtual void func(){ std::cout << val << std::endl;}
    void test(){func();}
    int val = 0;
};
class B: public A{
    B(){test();}
    virtual void func(){ std::cout << ++val << std::endl;}
    int val = 0;
};
A* pa = (A*)new B; pa->func(); 
```
在上面的代码中，new B时首先去调用A的构造函数此时第一次打印0；随后执行子类的构造函数，子类调用test函数，基类的test有去调用了func这一已经完成了重写的函数，满足多态条件，所以对val += 1后再打印1，最后就是通过pa指针调用，完成打印2的操作
### 动态绑定和静态绑定
对于不是多态调用条件的函数调用，在编译器就已经确定了函数调用地址（函数调用其实编译之后就是存在代码段的指令），这叫做静态绑定；而满足多态条件，在运行时去指向对象的虚函数表里面寻找的就是动态绑定。
### 虚函数表
* 基类的虚函数表存放了基类所有虚函数的地址，在继承体系中不同的类有各自的虚函数表，派生类的虚函数表和基类不同
* 一般来说，如果派生类继承的基类中有虚函数表指针，那么派生类就不会生成新的虚函数表指针，但是他继承的这个指针并不指向基类的虚函数表，而是指向自己这个派生类所共用的一张虚函数表，其中存放有基类的虚函数的地址、派生类完成重写后的虚函数的地址、派生类自己的虚函数的地址，所谓虚函数表指针，其实就是一个指向函数指针数组的指针
## 左值与右值
### “左”、“右”：
有一种说法是left与right的意思，但其实是locator value与read value的意思，即两者的核心区别是是否可以取到地址。对于临时变量、只读常量、匿名变量，这些具有常性的变量无法取出地址
，所以都是只读的，都是右值；而左值我们都能取到地址
### 左值与右值的转化
可以认为左值相比右值缺少了“常性”，想要用左值引用去引用右值，那么就可以使用const修饰的左值引用；如果想要用右值引用引用左值，那么就需要将左值交给move函数处理
### 移动构造和移动赋值
移动其实就是移动了资源，不再构造新的对象，而是将传过来的右值中的资源转移到目标中
### 类型分类
C++11后，右值和左值又进行了进一步分类，其中右值可以分为纯右值和将亡值
* 纯右值：字面常量、返回结果是字面量、不具名临时变量这些都是纯右值，比如"string", 1, a++等等
* 将亡值：将亡值是指返回右值引⽤的函数的调⽤表达式和转换为右值引⽤的转换函数的调⽤表达，例如move(x)、static_cast<X&&>(x)
<br>
泛左值可以分为左值和将亡值
