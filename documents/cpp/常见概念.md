## 基础概念
### 缺省参数
缺省参数需要注意的点有两个：
1. 缺省参数的声明和给定<br>
在函数声明中，缺省参数必须从右向左给定；使用函数时，参数必须从左向右依次传入，中间不允许出现跳跃情况
2. 缺省参数的出现位置<br>
在声明和定义分离的情况下，缺省函数只能出现在声明中，不能出现在定义中；但是不分离的情况下，因为函数的定义这时等价于声明，所以可以直接出现
### 函数重载
c++允许在同一作用域中出现同名函数，在形参不同时（可以时形参数目不同，也可以时形参类型不同），形成函数重载，表现出多态的行为。
<strong>需要注意的是，返回值不同不能作为判断依据，也很好理解，返回值时函数调用完的产物，但是不调用又不知道返回值的区别，编译器就犯难了</strong>

### 引用 
1. 引用其实就是给变量取了别名，对于引用对象，不会再为它开辟内存空间，其与被引用对象共享内存空间（可通过取地址查看），但是引用在使用时需要注意这些问题
* 引用在定义时必须初始化
* 一个对象可以有多个引用
* 与java不同，引用一旦初始化后，不能再改变指向
```cpp
int a = 10;
//int &b;
int &b = a;
int &c = a;
```
#### const 引用
1. 使用范围：对于常量，或者是像临时变量这种具有常性的变量，需要使用>const
2. 权限问题：权限可以缩小，但是不能放大，对于const引用，可以引用普通对象，但是普通引用不能引用常量
3. 指针和引用的关系:<br>
* 引用不开辟内存空间，但是指针需要开辟存放地址的空间
* 引用可以直接访问被引用对象，但是指针需要先解引用
* sizeof结果上，引用返回的是被引用对象的sizeof结果，而指针是该平台指针大小 
* 初始化问题：引用在定义时必须初始化，指针只是建议
* 改变指向问题：引用在确定引用对象后不能改变，但是指针在指向一个变量后可以随时改变指向对象
```cpp
int x = 10;
const int& y = x * 3;
const int& z = 10;
const int& m = 10;
const int& n = x;
```
### inline 关键字
对于inline修饰的函数，编译器会选择性的将其在被调用的地方展开，不再为该函数常见函数调用栈帧，类似与C语言的宏展开，不过需要注意的是，对于递归和代码量大的函数，编译器会给予忽略<br>
另外，内联函数不建议声明与定义分离，因为内联函数只是展开，不会开辟地址，所以在链接时会报错
### nullptr
* nullptr: c++中的nullptr是一种特殊类型的字面量，可以转化为其他任意的指针类型，但是不能够转化为整数
* NULL: C语言中的NULL是一个宏，可能是数字0，也可能是(void*)0，可以在stddef.h中查看，具体是哪个，取决于是否定义__cplusplus
* 综上，NULL对于c++这种提供了函数重载的语言来说是不好的（因为不能对NULL强制类型转换），例如以下代码
```cpp
void func(int* a)
{
    std::cout << "int*" << std::endl;
}

void func(int a)
{
    std::cout << "int" << std::endl;
}
```
使用NULL就可能会导致调用问题，甚至会可能直接编译报错，但是nullptr不能饮食类型转换为整形，所以就避免了这样的问题
## 类和对象
### 类域的概念
类定义的自己的作用域，外界访问需要通过‘：：’操作符，否则编译器默认不会到类域查找，只会在全局作用域；对于类中声明的函数，分离实现时，也需要指明类域。除此之外，类域也具有特殊的性质，就是编译器编译顺序，类域外，A函数在B函数前，但是A函数又调用了B函数，就会编译报错；类域中，查找顺序不再是简单的从上到下，而是整个类域都可以
### 类的实例化
* 所谓实例化，就是根据类定义中成员函数和成员变量的声明，分配相应的内存空间进行存储（成员函数有些特殊，这里不对成员函数的函数指针进行存储，后面还会提到，函数被编译后，形成一段指令，存储在代码段）
* 对于大小，其实和计算C语言的结构体没有特别大的区别，需要注意，对于空的类（没有成员函数和成员变量）以及只有成员函数的类，也会用一个字节来表示这个类的存在，不会不开辟内存空间
```cpp
class A
{
public:
    void f1()
    {
        std::cout << "f1" << std::endl; 
    }
    void f2()
    {
        std::cout << "f2" << std::endl;
        std::cout << _a << std::endl;
    }
private:
    int _a;
}

int main()
{
    A* a = nullptr;
    a->f1();
    a->f2();
    return 0;
}
```
* 以上程序，就会报错，原因是编译后，函数形成的指令可存储在代码段，但是并未实例化A，以为这没有对_a分配内存空间，这样在调用f2时就会段错误
### this指针
* 为了表明成员函数表明调用的是哪个实例化对象中的成员变量，c++引入了this指针，成员函数内部对成员变量的操作其实都是被这个隐式的this指针完成的，但是this指针不允许在形参和实参中出现，只可以在函数体中显示调用
* 对于非静态成员函数，每个成员函数都有自己的this指针，这个指针可以为空，不过不能直接赋空，可以强制类型转换
```cpp
class A
{
public:
    void f(int a)
    {
        this->a = a;
    }
private:
    int a;
}
```

### 构造函数
* 类名+参数列表，不写返回值，这就是构造函数
* 默认构造函数，包括三种
  1. 全缺省构造函数
  2. 无参构造函数
  3. 编译器自己生成的构造函数

总结起来其实就是不需要传参就能够使用的构造函数，但是默认构造函数只能存在一个，因为1、2如果自己实现了编译器就不会实现3，同时1、2也不能同时存在，因为存在调用二义性
* 对于内置类型，编译器生成的构造函数可以直接处理，但是对于自定义类型，则会去调用其默认构造函数，如果没有则报错
#### 初始化列表
* 无论是否显示实现初始化列表，都会先走初始化列表，然后才是构造函数函数体中的操作
* 每个成员变量在初始化列表中只能出现一次，并且对于引用、const成员变量、和没有默认构造的类类型变量，都必须在初始化列表初始化
* 初始化列表初始化顺序：只取决于声明的顺序，与初始化列表中的顺序无关
* 构造函数初始化顺序，有初始化列表优先按照初始化列表初始化，没有初始化列表，如果有缺省值，就用缺省值初始化，对于没有缺省值的，对于内置类型，可能初始化为默认值0，也可能初始化为随机值，对于自定义类型，如果没有默认构造，直接报错，对于引用类型和const成员变量，直接报错
### 析构函数
* ~+类名+() 就是析构函数，需要注意的是，对于内置类型，编译器自己生成的析构函数不做处理，因为这些栈上的内存会随函数栈帧的销毁而销毁；而自定义类型会调用他的析构函数
* 与构造函数相比，一个类只能有一个析构函数，如果没有申请资源，也可以不写析构函数
* 对于类中类，定义顺序在前的后析构
### 拷贝构造函数
* 是构造函数的重载，调用时对象还未实例化，参数要求：第一个参数必须是这个类的引用，后面还可以加参数，但是都必须有缺省值，否则虽然不会报错，但是编译器仍然会调用自己生成的拷贝构造函数
* 编译器默认生成的构造函数是浅拷贝，即只是简单的完成值的拷贝，对于指向资源的指针，我们还需要深拷贝，即开辟另一块相同的资源，再让新的指针指向这块资源，这就是我们需要自己实现拷贝构造的情况。
* 简单判断是否需要自己实现拷贝构造：查看是否有析构函数且还在内部显示释放资源，若没有，则使用默认即可
* 如果返回自定义类型的对象时，使用的是传值返回，那么会调用拷贝构造，使用传引用返回时，如果返回对象的声明周期和返回的函数相同，那么会造成“野引用”的问题
### 赋值运算符重载
#### 运算符重载
* operator+重载运算符，就构成了运算符重载；如果这不是成员函数，那么参数个数等于操作数个数，比如一元操作符需要一个参数，二元操作符需要两个参数；但是对于成员函数类型，参数的个数就比操作数个数少一个，因为第一个参数会默认传给隐式的this指针
* 前置++和后置++，两者的函数名是相同的，但是为了有所分别，规定后置++在参数列表（只是为了分别）加上一个int形参
* 规定，不能够重载原来不是操作符的符号
* 规定，不能够重载这些操作符'.*', 'sizeof', '::', '?:', '.'
* .*操作符表示通过对象调用成员函数指针
```cpp
class A
{
public:
    void func()
    {
        std::cout << "func" << std::endl;
    }
};
typedef void(A::*PF)(); //定义一个成员函数指针
int main()
{
    PF* pf = &A::func; //cpp规定，加上&才能够取到成员函数指针
    A a;
    (a.*pf)(); 
    return 0;
}
```
#### 赋值运算符重载
* 赋值运算符重载建议返回当前类型引用以完成连续赋值，同时传参时建议接受const引用，以满足临时对象和减少拷贝的需求
* 对于编译器自己生成的赋值运算符重载，进行的是浅拷贝，如果没有申请的资源，那么编译器生成的已经足够使用，对于需要自己实现的情况，判断方法和拷贝构造相同
* 赋值运算符重载如果不在类中显示实现，编译器就会自己生成，这意味这赋值运算符重载只能在类域中作为成员函数实现，否则自己的就会和编译器的发生冲突
* 实现细节，可以通过this指针进行判断，避免自赋值的情况
### 取地址运算符重载
#### const成员函数
1. 使用方法，在参数列表后加上const
2. 效果，const作用于this指针，让this指针从（假设日期类）Date* const 到const Date* this，表明该成员函数不得修改任何成员变量
#### 取地址运算符重载
1. 其实一般情况下编译器生成的已经足够使用，但是如果有一些需要，比如不希望外界拿到自己某个类的地址，就可以胡乱返回一个指针
2. 分为普通取地址重载和const取地址重载

### 类型转换
* 允许内置类型转换为自定义类型，需要在构造函数中进行实现，也可以将一种自定义类型转换为其他自定义类型
* 这种转换是隐式的，只要提供了相应的构造函数，但是如果不希望存在这种转换，只需在相应构造函数前加上explicit关键字即可
### 静态成员
* static修饰的成员变量和成员函数，都是一个类的静态成员
* 对于静态成员变量，不能在类中初始化，也不能通过在声明时给予初始值的方式进行初始化，只能在类外进行初始化
* 静态成员变量的存储：不存储在类中，存储在静态区中
* 静态成员函数可以访问静态成员变量，但是因为静态成员函数没有this指针，所以不能访问非静态成员，外界想要访问静态成员变量，通过::或者.就可以进行访问。
* 静态成员不属于某一个具体的实例化的类，一个类的所有非静态成员函数都可以访问这个类的所有静态成员变量和函数
* 静态成员也受到public、private、protected这些访问限定符的限制
### 友元
* 用来突破类域的限制，分为友元类和友元函数两种，可以放在类的任何位置
* 友元不具有传递性，是单向的，也可以是一对多的关系
### 内部类
* 将一个类的定义放在另一个类当中，这样被“包”起来的类就叫做内部类，它被默认为是外部类的友元类，能够访问外部类的所有成员函数和变量
* 定义内部类时，需要通过'::'突破外部类的类域来进行访问
### 匿名对象
* 声明周期只在当前行，主要使用于临时使用的场景，不需要对他取名字
比如下面的代码就是匿名对象的使用场景之一
```cpp
class A
{
public:
    A(int a = 0)
        : _a(a)
    {}
    int Solution()
    {}
private:
    int _a;
};
int main()
{
    //A a(); 这样写，编译器无法识别这是函数声明还是对象定义
    A (1).Soluiton();
    A().Solution();
    return 0;
}
```
## 内存管理
### 操作符new和delete
* c++内存管理主要通过new和delete两个操作符完成。
* new的工作过程是，先调用operator new这个全局函数来申请空间，之后在这块空间上调用构造函数，返回指向这块空间的指针；delete的工作过程是先在获得的空间上调用析构函数，随后通过operator delete来释放空间。当处理失败时，两个函数都会抛出异常，至于new[]和delete[]就是批量处理.
* 与malloc和free的区别
1. malloc和free是函数，new和delete是操作符
2. malloc只是申请空间，没有执行构造；free只是释放空间，没有调用析构
3. malloc的返回值使用需要从void*强转，而new可以直接使用
4. malloc失败，返回NULL，但是new抛异常
5. malloc要手动计算开辟空间的大小，而new可以根据后面的类型自动完成开辟大小的确定
* 对于堆的大小，只有操作系统决定，只能动态分配；栈的大小一般较小，也可以通过_alloca动态分配，不过不是通过free和delete释放
* 频繁的new和delete会导致堆的碎片化，但是栈不存在这个问题，因为栈是自动管理的
* new其实内部调用了malloc，delete其实内部调用了free，所以两者也需要头文件malloc.h支持，只不过一般这个头文件已经被其他各种文件包含了
### 定位new
* 其实定位new就是把一块已经申请的内存通过构造函数实例化出对象，在内存池中有应用
* 使用方法 new(ptr_to_mem)class_name(initializer_list)
示例如下
```cpp
class A
{
public:
    A(int a) : _a(a) {}
    ~A(){}
private:
    int _a;
};
int main()
{
    A* pa = (A*)malloc(sizeof(A));
    new(pa)A(10);
    pa->~A();
    operator delete(pa);
    return 0;
}
```
可以认为定位new就是把new和delete的工作交给自己手动分步完成
